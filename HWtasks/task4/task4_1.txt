Расширьте приложение FastAPI, добавив валидацию отзывов и запрет на использование нежелательных слов. Новая версия конечной точки /feedback должна принимать JSON с отзывом, проверять корректность и сохранять его.

1. Определите Pydantic модель с именем Feedback со следующими ограничениями:

name — строка, обязательное поле, от 2 до 50 символов
message — строка, обязательное поле, от 10 до 500 символов
2. Добавьте кастомную валидацию поля message, которая будет отклонять отзывы, содержащие недопустимые слова: "редиска", "бяка", "козявка" (в любых падежах). Если слово найдено — выбрасывайте ValueError с сообщением "Использование недопустимых слов".

3. Создайте POST-маршрут /feedback, который:

Принимает JSON по модели Feedback
При успехе сохраняет данные в список feedbacks
Возвращает JSON с сообщением: {"message": "Спасибо, {name}! Ваш отзыв сохранён."}
Пример запроса:

{
    "name": "Анна",
    "message": "Этот курс заставляет меня напрячься, но я все равно его пройду!"
}



Пример ответа:

{
    "message": "Спасибо, Анна! Ваш отзыв сохранён."
}



Пример некорректного запроса:

{
    "name": "А",
    "message": "тут сплошные редиски кругом, житья совсем не дают"
}
