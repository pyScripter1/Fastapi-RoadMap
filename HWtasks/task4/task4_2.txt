Усовершенствуйте систему обработки отзывов
Расширьте предыдущее решение, реализовав многоуровневую валидацию и интеграцию параметров запроса. Для успешного выполнения потребуется:

▸ Требования к моделям
Вложенная в фидбэк модель Contact:
Обязательное поле email с валидацией формата
Опциональное поле phone (только цифры, 7-15 символов)
Основная модель Feedback:
name: строка 2-50 символов
message: строка 10-500 символов
▸ Бизнес-логика
Автоматическая проверка message на запрещённые слова ("редиска", "бяка", "козявка" в любых формах)
Реализация параметра запроса is_premium:
При значении true — добавление в ответ фразы: "Ваш отзыв будет рассмотрен в приоритетном порядке"
Пример ответа: "Спасибо, {name}! ... [приоритетная фраза]"
Примеры реализации
▸ Стандартная обработка
POST /feedback
{
    "name": "Артём",
    "message": "Понравилась структура курса: теория сразу закрепляется практикой",
    "contact": {
        "email": "artem@study.ru"
    }
}


Ответ:

{
    "message": "Спасибо, Артём! Ваш отзыв сохранён."
}


▸ Приоритетная обработка
POST /feedback?is_premium=true
{
    "name": "Виктория",
    "message": "Подробные примеры из реальных проектов помогают понять нюансы разработки",
    "contact": {
        "email": "vika@protonmail.com",
        "phone": "79051234567"
    }
}


Ответ:

{
    "message": "Спасибо, Виктория! Ваш отзыв сохранён. Ваш отзыв будет рассмотрен в приоритетном порядке."
}


Критические точки валидации:
Сообщение ровно 500 символов должно проходить проверку
Номер телефона "7905123" (7 цифр) — допустим
Номер телефона "7905123a" — должен вызывать ошибку
Сообщение "Этот козявка испортил весь курс, это возмутительно и неприемлемо!" — должно блокироваться
